Git & GitHub

Concise notes covering remote setup, pushing/pulling, forks vs clones, PRs, issues, and docs.

Git basics

git remote add links your local repo to an online repo (commonly named origin), enabling push/pull. git push uploads commits to the remote; git pull downloads and merges remote changes into your local branch.

Clone downloads a repository to your machine. Fork creates your own copy of someone else's repo on GitHub to work independently.

Pull Requests & Code Review

A Pull Request (PR) is how you propose merging one branch into another (usually into main). Teammates review changed lines, add comments, and request fixes. Push new commits to your branch to update the PR; once approved it’s merged and the branch can be deleted.

Issues, Comments & Task Assignment

Issues track bugs, features, or questions with a title, description, labels, assignees, and comments. Use comments for discussion and screenshots. Assign tasks to people to avoid duplicated work and include Markdown task lists to track progress.

  • Create Issue → assign → fix in branch → open PR → merge → close Issue.

Documentation & README

A good README contains a clear title, short intro, installation & usage steps, contributor guidelines, optional demo/diagram, and known issues. Project docs (a docs folder, API refs, contribution guides, license) help others use and contribute to your project.

Markdown essentials

Markdown formats docs and content on GitHub. Common features:

  • Headings: # H1, ## H2, ### H3
  • Bold/Italic: **bold**, *italic*, ***both***

Portfolio note

A portfolio should present who you are, highlight your best projects (with links), list skills, and include contact info. Well-documented GitHub projects make excellent portfolio pieces.
