Git Basics

Getting into Git felt a bit overwhelming at first, but things clicked after using The Odin Project. Below are the core commands and a short explanation of how they fit together.

Topics

  • git init
  • git status
  • git add
  • git commit
  • git log
  • Commit messages

Summary

Use git init to start a repository. Run git status to see which files are staged, unstaged, or untracked. Stage changes with git add (file, folder, or git add .) and save them with git commit -m "message". Check history with git log. Clear, present-tense commit messages keep the project easy to follow. 

git init
git status
git add <file>
git commit -m "describe what you changed"
git log
