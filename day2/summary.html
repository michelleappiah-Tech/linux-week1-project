Git branching & merging

Topics: git branch, git checkout, git switch, merge, resolving conflicts

As I continued practicing Git, I started exploring how branches work and realized they make managing different parts of a project much easier. A branch is a separate line of development where I can work on features, fixes, or ideas without touching the main project.

Common commands

  • git branch
  • git checkout
  • git switch
  • git switch -c
  • git merge
  • resolving merge conflicts

Practices & notes:

Use git branch to list/create branches. Switch branches with git switch <name> (preferred) or git checkout <name> (older). Create+switch: git switch -c <name>. Rename or delete branches with git branch -m and git branch -d (or -D to force delete).

To merge: switch to the target branch (e.g. main) and run git merge <feature-branch>. Merges can be fast-forward or create a merge commit. Merge conflicts require manual resolution — Git will mark conflicted files; after fixing them, run git add and git commit to complete the merge.

# create a branch
git branch feature-xyz

# create and switch
git switch -c feature-xyz

# switch (preferred)
git switch feature-xyz

# old way to switch
# git checkout feature-xyz

# merge into main
git switch main
git merge feature-xyz

# after resolving conflicts
git add <file>
git commit -m "resolve merge conflicts"
Understanding branching and merging makes version management much easier — conflicts are just decisions waiting to be made.
